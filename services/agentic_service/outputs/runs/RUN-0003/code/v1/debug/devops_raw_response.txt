services:
  mongodb:
    image: mongo:latest

    ports:
      - "27017:27017"
    volumes:
      - mongo_data:/data/db

  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: autoforge_backend
    ports:
      - "9000:9000"
    environment:
      NODE_ENV: development
      MONGO_URI: mongodb://mongodb:27017/autoforge
    depends_on:
      - mongodb
    volumes:
      - ./backend:/app/backend
      - /app/backend/node_modules
    restart: always

  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    container_name: autoforge_frontend
    ports:
      - "5173:5173"
    environment:
      VITE_API_URL: http://localhost:9000
    depends_on:
      - backend
    volumes:
      - ./frontend:/app/frontend
      - /app/frontend/node_modules
    restart: always

# AutoForge E-commerce Demo

## Project Overview
AutoForge is a modular monolith e-commerce demonstration built using the MERN stack (MongoDB, Express, React, Node.js). The application simulates core e-commerce functionalities including product catalog management, user authentication, and shopping cart operations.

The architecture is designed for modularity, separating the concerns into distinct backend (API) and frontend (Client) services, while utilizing MongoDB as the persistent data store.

## Prerequisites
Before running the application, ensure you have the following installed:
*   Docker Engine
*   Docker Compose

## How to Run with Docker Compose (Recommended)
This method sets up and runs all services (MongoDB, Backend, Frontend) in isolated containers, managing dependencies and networking automatically.

1.  **Clone the repository:**
    git clone <repository-url>
    cd autoforge
2.  **Build and run services:**
    docker-compose up --build
3.  **Access the application:**
    The application will be accessible at `http://localhost:5173`.

## Manual Setup Instructions

If you prefer running services directly on your host machine (requires manual dependency management):

### 1. Backend (Node.js/Express)
1.  Navigate to the backend directory: `cd backend`
2.  Install dependencies: `npm install`
3.  Set up environment variables (see below).
4.  Run the server: `npm run dev` (or `node server.js`)

### 2. Frontend (React/Vite)
1.  Navigate to the frontend directory: `cd frontend`
2.  Install dependencies: `npm install`
3.  Set up environment variables (if needed).
4.  Run the development server: `npm run dev`

## Environment Variable Setup
The application requires a `.env` file in the root directory (or within the respective service directories if running manually).

**Required Variables:**
| Variable | Scope | Description | Example Value |
| :--- | :--- | :--- | :--- |
| `MONGO_URI` | Backend | Connection string for MongoDB. | `mongodb://localhost:27017/autoforge` |
| `PORT` | Backend | Port the Express server runs on. | `9000` |
| `VITE_API_URL` | Frontend | Base URL for API calls (should match backend port). | `http://localhost:9000` |

## Useful URLs
*   **Frontend Client:** `http://localhost:5173`
*   **Backend API:** `http://localhost:9000`
*   **Database:** `mongodb://localhost:27017`
